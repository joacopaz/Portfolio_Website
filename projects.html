  • Clutch Agents

    Description

    Multi-tenant AI agent platform built around two parallel runtime paths: an OpenClaw-powered agent that runs on the platform's own infrastructure, or a remotely hosted agent that delegates the full agent loop to a provider's managed service (Anthropic Managed Agents, OpenAI Agents, and similar offerings). Both paths sit behind a single normalized contract, so each runtime behaves identically to the rest of the system, and adding a new provider (Meta, others) is a matter of dropping in an adapter without touching shared code.

    Each agent runs paired with its own egress proxy, an isolated persistence volume, and per-agent encryption keys. The platform takes a security-first approach to ingress and egress: a reverse proxy generates routing rules from the fleet database to gate inbound traffic, while every agent gets an authorized-domain allowlist on the way out plus domain-scoped credentials, so a prompt injection that hijacks an agent's intent still cannot leak secrets to an unintended destination. A React and Node control plane handles fleet provisioning, customer auth, integrations, secrets, and live config push to each agent's filesystem. The whole system is shaped as a multi-agent orchestration layer for running complex, long-running workflows under a strict security boundary.

    Objective

    Treat every agent as untrusted. Secrets are injected at runtime by a host-side dispatcher and never enter the agent's process, the proxy enforces per-agent domain allowlists on every outbound call, and customers can wire over 2,000 one-click integrations (Gmail, Slack, Calendars, and the rest) without exposing credentials to the agent itself. Webhooks support bring-your-own-API alongside auto-detected signing for Stripe, Slack, Shopify, GitHub, and Meta from a single pasted secret.

    This is a proprietary product and the running platform is not publicly accessible. Happy to share a live walkthrough on request.

  • Google AI Edge Contribution

    Description

    Objective

  • Ls Operations Map

    Description

    Full-stack React web application using Firebase to manage the Database and Google Cloud Functions with ExpressJS to manage the API and Backend operations.

    Objective

    The client requested a web app that managed all the company's channels information with an Authentication system, a custom backend API to perform special Admin functions, a Role system, an Admin command overlay, custom DB modelling and the ability to enhance/develop/customize the project without hard coding, just based on Admin roles. The requirements were to minimize server costs, allow the dynamic edition of the whole database and an admin dashboard to control versions/rollback events.

    These were achieved using a version-control system (a local stored DB that only reads the online DB on a need-to-know basis, only checking if there is a new version of the online one and updating specifically the lines to update, to avoid extra reads/writes and minimize server costs).
